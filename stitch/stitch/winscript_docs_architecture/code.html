Enterprise Automation Framework

WinScript bridges the gap between high-level AI reasoning and low-level Windows API execution through a secure, structured pipeline optimized for the Model Context Protocol (MCP).

security Sandboxed Execution

Commands are validated against a strict allow-list of processes and subdirectories, preventing unauthorized lateral movement within the host OS.

verified_user Audit Logging

Every interaction—from the raw MCP request to the subprocess return code—is captured in an encrypted local telemetry log for compliance.

sync Protocol State

Maintains a stateful session between the Agent and the Win32 application tree, ensuring stable element discovery during multi-step tasks.

Visual Feedback Pipeline

WinScript implements the Claude Computer Use pattern for visual perception. Instead of sending raw binary data, the server captures the current UI context and streams it as a structured Base64 encoded block.
